# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.6.x   | :white_check_mark: |
| < 1.6   | :x:                |

## Reporting a Vulnerability

We take the security of LOD Manager seriously. If you believe you have found a security vulnerability, please report it to us as described below.

**Please do NOT report security vulnerabilities through public GitHub issues.**

Instead, please report them via [GitHub Security Advisories](https://github.com/torrua/LOD_manager/security/advisories/new).

You should receive a response within 48 hours. If for some reason you do not, please follow up via the same channel.

### What to Include

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested fixes (optional)

### What to Expect

- **Within 48 hours**: Acknowledgment of your report
- **Within 7 days**: Initial assessment and severity determination
- **Within 30 days**: Fix release or mitigation plan
- **After fix**: Public disclosure and credit (if desired)

## Security Best Practices for Users

- Always download LOD Manager from the [official GitHub releases page](https://github.com/torrua/LOD_manager/releases)
- Verify file checksums when available
- Keep the application updated to the latest version
- Do not open dictionary files from untrusted sources
