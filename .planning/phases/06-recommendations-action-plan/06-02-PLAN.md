<action>
Create risk mitigation playbook for all High and Medium severity risks from Phase 5.

**For each risk define:**

1. **Risk description** — what can go wrong
2. **Likelihood** — Low/Medium/High (how often it might happen)
3. **Impact** — Low/Medium/High (how bad it is when it happens)
4. **Mitigation strategy** — how to prevent it
5. **Fallback plan** — what to do if it happens anyway
6. **Monitoring** — how to detect it early
7. **Owner** — which part of the codebase is responsible

**Risks to cover (from ASSESSMENT-03-STABILITY.md):**

### High Severity

1. Transaction panic in import (`import.rs:113` — `.unwrap()`)
2. Partial state on open failure (`lib.rs:48-60`)

### Medium Severity

3. FTS rebuild memory concern (`db.rs:800-813`)
4. No migration rollback (`db.rs:150-183`)
5. Android file size not validated
6. Error categorization missing (generic strings)

**Format:**

```markdown
# Risk Mitigation Playbook

## Risk Register with Mitigations

### Risk 1: [Title]

- **Severity:** [High/Medium]
- **Likelihood:** [Low/Medium/High]
- **Impact:** [Low/Medium/High]
- **Source:** [assessment:line]

**Description:** [what can go wrong]

**Mitigation Strategy:**

1. [step 1]
2. [step 2]

**Fallback Plan:**

- [what to do if it happens]

**Monitoring:**

- [how to detect early]

**Code Changes Required:**

- [file:line] — [what to change]
```
