<name>Task 1: Prioritized Recommendations (REC-01)</name>
<files>.planning/phases/06-recommendations-action-plan/RECOMMENDATIONS.md</files>
<action>
Create prioritized recommendation list synthesizing all Phase 5 findings.

**Read all Phase 5 assessments** and extract every finding that requires action.

**Categorize by priority:**

### P0 — Must Fix (stability/correctness)

Items that could cause data loss, panics, or incorrect behavior.

### P1 — Should Fix (reliability/confidence)

Items that improve reliability but aren't critical. Missing test coverage for core features.

### P2 — Nice to Have (maintainability)

Items that improve developer experience but don't affect users.

### P3 — Future (strategic)

Items to consider if the app grows significantly.

**For each recommendation include:**

- Title
- Source (which Phase 5 assessment)
- Problem description
- Recommended action
- Effort estimate (XS/S/M/L/XL)
- Impact estimate (Low/Medium/High)
- Files affected

**Format:**

```markdown
# Recommendations: Prioritized Action List

## P0 — Must Fix

### 1. [Title]

- **Source:** [assessment name]
- **Problem:** [description]
- **Action:** [what to do]
- **Effort:** [XS/S/M/L/XL]
- **Impact:** [Low/Medium/High]
- **Files:** [file paths]

## P1 — Should Fix

...

## P2 — Nice to Have

...

## P3 — Future

...

## Summary Table

| #   | Priority | Title | Effort | Impact | Files |
| --- | -------- | ----- | ------ | ------ | ----- |
| 1   | P0       | ...   | S      | High   | ...   |
```

</action>
